🤖 Machine Learning Models

🌲

Random Forest

An ensemble of decision trees trained with bagging. Robust to outliers and handles non-linear relationships well. Used as a strong baseline for all venue types.

n_estimators=200 max_depth=10 n_jobs=-1
📈

Gradient Boosting

Sequential ensemble that corrects errors from previous trees. Excellent at capturing complex patterns. Uses a low learning rate with many estimators for precision.

n_estimators=200 lr=0.05 max_depth=4
📐

Ridge Regression

L2-regularised linear model. Fast, interpretable, and surprisingly strong when features are well-engineered. Often wins on smaller or cleaner datasets.

alpha=1.0 linear L2 penalty
🗳️

Voting Ensemble

Soft average of all three models. Only deployed if its R² beats every individual model. Reduces variance and improves generalisation across venue types.

VotingRegressor averaging auto-selected
🏆

Auto Model Selection

All four models are trained per venue type. The one with the highest R² on a held-out 20% test split is automatically saved and used for live predictions.

test_size=0.2 random_state=42 best R² wins
📊

Evaluation Metrics

Every model is evaluated on three metrics to give a complete picture of performance across all venue types.

R² Score MAE RMSE

⚙️ Data Pipeline

1

Data Generation / Upload

Train on synthetic data (2,000 rows per venue) generated with realistic distributions, or upload your own CSV dataset.

→
2

Label Encoding

Categorical features (meal type, season, day of week) are label-encoded per column with unseen-label handling.

→
3

Standard Scaling

All features are standardised (zero mean, unit variance) using sklearn's StandardScaler before model training.

→
4

Train / Test Split

80% training, 20% test split with a fixed random seed (42) for reproducibility across all venue types.

Lightweight web framework serving the UI and REST API endpoints.
